### Phase Handoffs

| Handoff | Status | Evidence |
|---------|--------|---------|
| Phase 01 → 02 | **WIRED** | Event page (page.tsx L113) calls `getSession(id)`; null session → `<ParticipantActions>` + `<JoinFlow>`; valid session → `<AvailabilityCTA>`. `window.location.reload()` after join triggers server re-render to pick up new session cookie. |
| Phase 02 → 03 | **WIRED** | All 3 session issuers (join, verify-pin, magic-link/consume) import `SESSION_COOKIE` constant from `auth.ts`. `POST /api/availability` calls `getSession()` from same module → 401 without valid session. Single-sourced constant, no hardcoded cookie strings. |
| Phase 03 → 04 | **WIRED** | Phase 4 aggregates Phase 3 `availability` rows via `GROUP BY slotStart, COUNT(*)`. `heatmapMap` + `peakCount` + `bestSlot` computed server-side and passed to `<HeatmapGrid>`. `slotColor(count, peakCount)` from `heatmap-color.ts` applied per cell inline. |
| Phase 04 → 05 | **WIRED** | `deleteEvent` Server Action reads/deletes same `timely_creator_{id}` cookie as `confirmTime`. CASCADE DELETE covers all 5 child tables via FK `onDelete: 'cascade'`. `timely_flash_toast_{uuid}` cookie → `FlashToast` in root layout fires Sonner toast after redirect. |

### Integration Warnings (Non-Fatal)

**WARNING 1: Post-confirm UI relies on implicit RSC re-render (HEAT-06)**

`confirm-time-sheet.tsx` lines 41–43: After `confirmTime` Server Action succeeds, the sheet closes without an explicit `router.refresh()`. The `revalidatePath('/e/${eventId}')` in the Server Action triggers an implicit Next.js App Router RSC re-render. Confirmed working in Playwright tests (04-05-SUMMARY.md Test 5). Idiomatic App Router pattern — not a bug.

**WARNING 2: SECR-03 expiry approximation**

`api/events/route.ts` L69: `expiresAt = addDays(new Date(), 37)` at creation. Requirement says "30 days after last candidate date." Functionally adequate for typical near-future scheduling. Code comment documents this as a known approximation.
