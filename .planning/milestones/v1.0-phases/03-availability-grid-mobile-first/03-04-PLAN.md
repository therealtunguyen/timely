<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: Human verification — drag-to-paint, persistence, timezone, mobile touch targets</name>
  <action>
    Verify the complete availability grid flow across 6 test scenarios using Chrome DevTools mobile emulation (iPhone 12 Pro, 390px) and normal desktop.
  </action>
  <what-built>
    Complete availability grid: vaul drawer with drag-to-paint grid, timezone auto-detect, explicit save button, auto-save on close, slot persistence via GET/POST /api/availability.
  </what-built>
  <how-to-verify>
    Use Chrome DevTools mobile emulation (iPhone 12 Pro, 390px) for touch tests, and normal desktop for desktop tests.

    **Test 1 — Touch drag-to-paint + CTA text (GRID-01, GRID-02, MOBI-04):**
    1. Open the event page in Chrome DevTools with iPhone 12 Pro emulation enabled
    2. Join as a new participant (name + PIN) — complete the Phase 2 flow
    3. EXPECTED on reload: CTA button reads "Mark your availability" (not "Edit") — this is a new participant with no prior submission
    4. Tap "Mark your availability" — the grid drawer should open
    5. In the grid, drag across multiple time cells in a single gesture
    6. EXPECTED: cells paint orange as your finger moves; the page does NOT scroll or bounce; the drawer does NOT dismiss
    7. Save, close, and reopen — EXPECTED: CTA button now reads "Edit your availability" (participant has submitted)
    8. PASS if: CTA shows correct text per scenario AND multiple cells paint in one drag gesture without scroll interference

    **Test 2 — Save and return (GRID-07, GRID-08):**
    1. With the grid open and some cells painted from Test 1, tap "Save availability"
    2. EXPECTED: Sonner toast "Availability saved" appears; you remain in the grid
    3. Close the drawer (X button or drag handle)
    4. Open the drawer again
    5. EXPECTED: Sonner toast "Loaded your previous availability" appears; previously marked cells are pre-filled
    6. PASS if: saved slots appear on re-open

    **Test 3 — Timezone display (TIME-02, TIME-03, TIME-04):**
    1. Open the grid drawer
    2. EXPECTED: time labels in the grid match your local timezone (not UTC), shown in 12-hour format
    3. Tap "Wrong timezone?" link
    4. EXPECTED: a timezone selector dropdown appears with a warning about clearing selections
    5. Select a different timezone (e.g., "America/Los_Angeles" if you're in Eastern time)
    6. EXPECTED: grid time labels immediately update; any painted selections are cleared
    7. PASS if: labels update and selections clear on timezone change

    **Test 4 — 14-day date-range (GRID-05):**
    1. Create a new event using a date range spanning 14 days
    2. Open that event page and open the grid drawer
    3. EXPECTED: grid renders 14 date columns; horizontal scroll works; time column stays visible (sticky) during horizontal scroll
    4. PASS if: 14 columns render without layout breakage; time column is visible after scrolling right

    **Test 5 — 44px touch target check (GRID-04):**
    1. In Chrome DevTools, open the grid drawer
    2. Right-click any grid cell → Inspect
    3. In the Elements panel, check the computed height of a grid cell
    4. EXPECTED: computed height >= 44px
    5. PASS if: all spot-checked cells measure at least 44px height

    **Test 6 — Auto-save on close:**
    1. Open the grid, paint some cells, do NOT tap Save
    2. Close the drawer using the X button or drag handle
    3. Re-open the drawer
    4. EXPECTED: "Loaded your previous availability" toast and previously painted cells are pre-filled
    5. PASS if: auto-save fires on drawer close and data persists
  </how-to-verify>
