Create src/components/availability/heatmap-grid.tsx. This is a 'use client' component because it reads from heatmapStore for tap-a-name dim state.

**Props:**
```typescript
interface HeatmapGridProps {
  dates: string[]                     // ISO date strings "YYYY-MM-DD" — same as AvailabilityGrid
  dayStart: number                    // Hour 0-23
  dayEnd: number                      // Hour 0-23
  timezone: string                    // IANA tz for label rendering (event's timezone)
  heatmapMap: Record<string, number>  // slotKey (UTC ISO) -> participant count
  peakCount: number                   // Highest count across all slots
  totalParticipants: number           // Total participant count (for aria-labels)
  participantSlots: Record<string, string[]>  // participantName -> UTC ISO slot keys (for intersection)
  ownSlots: string[]                  // UTC ISO slot keys the current viewer has marked (empty array if unauthenticated)
}
```

**Implementation:**

1. Import: `useHeatmapStore` and `ParticipantSlots` from heatmap-store, `slotColor` from heatmap-color, `GridCell` from grid-cell, `fromZonedTime`, `toZonedTime`, `format` from date-fns-tz, `parseISO`, `format as formatDate` from date-fns.

2. Convert `participantSlots` prop (Record<string, string[]>) to `ParticipantSlots` (Record<string, Set<string>>) inside the component body — once at render, not per-cell:
   ```typescript
   const participantSlotsMap: ParticipantSlots = Object.fromEntries(
     Object.entries(participantSlots).map(([name, slots]) => [name, new Set(slots)])
   )
   ```

3. Get intersection state from store:
   ```typescript
   const { selectedNames, intersectionSlots } = useHeatmapStore()
   const hasSelection = selectedNames.size > 0
   const intersected = hasSelection ? intersectionSlots(participantSlotsMap) : new Set<string>()
   ```

4. Build slot rows using the same pattern as AvailabilityGrid — reuse the same `generateSlotKey` / `buildSlotRows` logic:
   ```typescript
   function generateSlotKey(date: string, hour: number, minute: number, tz: string): string {
     const localStr = `${date}T${String(hour).padStart(2,'0')}:${String(minute).padStart(2,'0')}:00`
     return fromZonedTime(localStr, tz).toISOString()
   }
   ```
   Build rows: `for hour in dayStart..dayEnd, for minute in [0, 30]`.

5. Build ownSlotsSet: `const ownSlotsSet = new Set(ownSlots)` — O(1) lookup per cell.

6. Render: same grid layout as AvailabilityGrid — CSS grid with `gridTemplateColumns: '80px repeat(N, minmax(64px, 1fr))'`, sticky time labels, date headers.

7. For each cell, compute:
   - `const count = Number(heatmapMap[cellKey] ?? 0)` — CRITICAL: coerce to number (count() can return string from Postgres via Neon driver)
   - `const color = slotColor(count, peakCount)`
   - `const dimmed = hasSelection && !intersected.has(cellKey)`
   - `const isOwn = ownSlotsSet.has(cellKey)`
   - Build time label for aria: e.g. "10:00–10:30 AM"
   - Build date label for aria: e.g. "Tuesday March 15"

8. Render `<GridCell>` in heatmap mode:
   ```tsx
   <GridCell
     key={cellKey}
     slotKey={cellKey}
     heatmapColor={color}
     isOwn={isOwn}
     dimmed={dimmed}
     count={count}
     totalParticipants={totalParticipants}
     timeLabel={timeLabel}
     dateLabel={dateLabel}
   />
   ```

9. The grid container should NOT have `onPointerDown` / `onPointerMove` handlers (read-only — no painting). Add `data-vaul-no-drag` to prevent vaul interference if rendered inside a sheet.

10. Empty cells (count=0): slotColor(0, peak) returns HEATMAP_LIGHT (#FAF7F4) — the off-white background visually distinguishes them from peak slots per the locked decision.

The outer wrapper: `<div className="overflow-x-auto">` wrapping the CSS grid div — matches AvailabilityGrid's scroll pattern.

**File 2: src/components/availability/participant-list.tsx**

Mark 'use client' — it reads from heatmapStore and handles toggle clicks.

Props:
```typescript
interface Participant {
  name: string
  submittedAt: Date | null
}

interface ParticipantListProps {
  participants: Participant[]
}
```

Implementation:
1. Import `useHeatmapStore` from heatmap-store.
2. Sort participants: responded first (submittedAt not null), then not-yet-responded. Within each group: alphabetical by name.
   ```typescript
   const sorted = [...participants].sort((a, b) => {
     const aResponded = a.submittedAt != null ? 0 : 1
     const bResponded = b.submittedAt != null ? 0 : 1
     if (aResponded !== bResponded) return aResponded - bResponded
     return a.name.localeCompare(b.name)
   })
   ```
3. Get `selectedNames` and `toggleName` from useHeatmapStore.
4. Render each participant as a tappable chip/button:

```tsx
<section className="space-y-2">
  <h2 className="text-sm font-medium text-[#6B6158] uppercase tracking-wide">
    {participants.length} {participants.length === 1 ? 'response' : 'responses'}
  </h2>
  <div className="flex flex-wrap gap-2">
    {sorted.map(p => {
      const isSelected = selectedNames.has(p.name)
      const hasResponded = p.submittedAt != null
      return (
        <button
          key={p.name}
          onClick={() => toggleName(p.name)}
          disabled={!hasResponded}  // Can't filter on someone who hasn't responded
          className={cn(
            "px-3 py-1.5 rounded-full text-sm font-medium transition-colors",
            isSelected
              ? "bg-[#1C1A17] text-white"
              : hasResponded
                ? "bg-[#F3EFE9] text-[#1C1A17] hover:bg-[#E5DDD4]"
                : "bg-[#F3EFE9] text-[#9D9086] cursor-default opacity-60"
          )}
          aria-pressed={isSelected}
          aria-label={`${p.name}${hasResponded ? ' — responded' : ' — no response yet'}${isSelected ? ', selected' : ''}`}
        >
          {p.name}
          {hasResponded && <span className="ml-1.5 text-[#9D9086] text-xs">✓</span>}
        </button>
      )
    })}
  </div>
</section>
```

Not-yet-responded participants: shown as disabled chips (greyed, no tap action) — they have no availability data to filter on.

Import `cn` from `@/lib/utils`.
