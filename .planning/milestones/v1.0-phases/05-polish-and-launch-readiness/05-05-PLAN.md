**Modify `src/components/availability/heatmap-grid.tsx`:**

The heatmap grid is a read-only data grid. It needs the ARIA grid pattern: `role="grid"` → `role="row"` → `role="gridcell"` / `role="rowheader"` / `role="columnheader"`.

Apply these changes to the existing grid structure:

1. On the outer grid container `<div>` (currently just `data-vaul-no-drag className="select-none" style={{display:'grid',...}}`), add:
   ```
   role="grid"
   aria-label="Availability heatmap"
   aria-rowcount={slotRows.length + 1}
   aria-colcount={dates.length + 1}
   ```

2. The header row is currently two separate fragments (the corner cell + date headers mapped individually). Wrap them in a single `<div role="row">`:
   ```tsx
   <div role="row">
     <div role="columnheader" className="sticky left-0 z-20 bg-[#FAF8F5]" aria-label="Time" />
     {dates.map(date => (
       <div
         key={date}
         role="columnheader"
         aria-label={format(parseISO(date), 'EEEE, MMMM d')}
         className="text-center text-xs font-medium text-[#6B6158] py-2 border-b border-[#E5DDD4] whitespace-pre-line"
       >
         {format(parseISO(date), 'EEE\nMMM d')}
       </div>
     ))}
   </div>
   ```

3. Each slot row (currently `<React.Fragment key={slotKey}>`) becomes `<div role="row" key={slotKey} aria-rowindex={rowIdx + 2}>` where `rowIdx` is the index from `slotRows.map((slot, rowIdx) => ...)`. Update the `.map` call to destructure the index.

4. The sticky time label cell (currently a plain `<div>`) gets `role="rowheader"`.

5. The `<GridCell>` already has `role="gridcell"` in heatmap mode (confirmed in existing code) and receives `aria-label` via the `ariaLabel` computed prop. Add `tabIndex={0}` to heatmap-mode GridCell in `grid-cell.tsx` so keyboard users can Tab through cells and have their aria-label read. BUT — for a 14-day grid with 24+ time slots, that's 336+ tab stops. Instead, apply `tabIndex={-1}` to all cells (so they're focusable via programmatic focus but not in tab order), except the first cell of each row gets `tabIndex={0}`. This is done in HeatmapGrid by passing a `tabIndex` prop to GridCell and checking if `colIdx === 0`:

   In HeatmapGrid's cell render:
   ```tsx
   {dates.map((date, colIdx) => {
     // ...
     return (
       <GridCell
         key={`${date}-${hour}-${minute}`}
         slotKey={cellKey}
         heatmapColor={color}
         isOwn={isOwn}
         dimmed={dimmed}
         count={count}
         totalParticipants={totalParticipants}
         timeLabel={timeLabel}
         dateLabel={dateLabel}
         tabIndex={colIdx === 0 ? 0 : -1}
       />
     )
   })}
   ```

   And in `src/components/availability/grid-cell.tsx`, add `tabIndex?: number` to the `GridCellProps` interface and apply it on the heatmap `<motion.div>`:
   ```tsx
   // In interface GridCellProps:
   tabIndex?: number

   // In heatmap motion.div:
   tabIndex={tabIndex}
   ```

**Modify `src/components/availability/availability-grid.tsx`:**

The edit grid is interactive (drag-to-paint). The ARIA treatment is lighter — it's not a data grid with semantic cells, it's a canvas-like interaction surface. Add `role="grid"` and `aria-label` on the container for orientation, and `role="row"` wrappers around each Fragment row. The individual cells already have `data-slot-key` but no ARIA roles in edit mode. Add `role="gridcell"` to GridCell in edit mode by:

1. On the outer drag container div, add `role="grid" aria-label="Availability grid — drag to mark your available times"`.
2. Wrap each `<React.Fragment key={slotKey}>` with `<div role="row">` instead. Since grid layout is CSS-driven (not table-driven), using a `<div role="row">` wrapper that's also part of the CSS grid is tricky — it would break the grid layout because the wrapper becomes a grid item itself. The correct approach for a CSS grid is to NOT add row wrappers (CSS grid flattens items regardless of wrappers) but instead use `aria-rowindex` on each cell directly.

   For the edit grid, a simpler approach works: add `role="gridcell"` to edit-mode GridCell in `grid-cell.tsx`. The edit-mode `<div>` in GridCell currently has no role — add `role="gridcell"` and an `aria-label` for the slot key in a human-readable form. Pass a `slotLabel` prop for the accessible label:

   Actually, the edit grid cells don't carry time/date context in an accessible way yet. The simplest correct approach: apply `role="grid"` on the container and `aria-label` explaining the interaction. Individual cells in edit mode don't need full ARIA cell roles for this app's scope — the grid role on the container is sufficient for assistive technology to announce "grid" context. The key accessibility requirement is keyboard navigation through the create flow, not full ARIA data grid for the edit canvas.

   Apply only: `role="grid" aria-label="Availability grid — drag or tap to mark your available times"` on the container div. This is sufficient for the edit grid scope.

2. Add `onKeyDown` handler for Enter and Space:
   ```tsx
   onKeyDown={isInteractive ? (e) => {
   ```
