  <what-built>
Complete Phase 4 heatmap and results view:
- Heatmap overlay on event page: slot colors interpolated from #FAF7F4 (empty) to #7C4A0E (peak) based on participant count
- BestTimeCallout always visible above the grid — shows top slot or "Waiting for responses"
- ParticipantList with tap-a-name multi-select: tapping names dims non-intersection slots
- Creator confirm-time flow: tapping callout opens vaul bottom sheet → "Confirm this time" locks the event
- Confirmed state: orange banner showing locked time; availability CTA hidden
  </what-built>
  <how-to-verify>
**Setup:** You need an event with at least 3 participants who have saved availability. If none exists, create a test event and have 3 browser tabs (incognito + private + normal) join with different names and mark different availability slots so there's overlap.

**Test 1 — Heatmap displays (HEAT-01, HEAT-02)**
1. Open the event page in a browser.
2. Verify: a heatmap grid is visible with cells in varying shades of amber/brown.
3. Verify: slots where more people are free appear darker than slots where fewer are free.
4. Verify: empty slots (zero people) appear as off-white (#FAF7F4), not orange.
5. Verify: the color scale uses warm amber/brown tones — no red or green anywhere.
Expected: Grid looks like a temperature map. The darkest cells should be noticeably darker amber-brown.

**Test 2 — Participant list and response status (HEAT-03)**
1. Check the participant list section below/near the grid.
2. Verify: participants who have saved availability are shown first (with checkmark).
3. Verify: participants who joined but have not saved are shown after (greyed out, not tappable).
Expected: Responded participants listed first, alphabetically within their group.

**Test 3 — Tap-a-name highlights slots (HEAT-04)**
1. Tap one participant's name in the list.
2. Verify: their name chip changes to dark background (selected state).
3. Verify: grid cells dim to near-white EXCEPT for slots where that person is free (those stay bright).
4. Tap a second participant name simultaneously.
5. Verify: the grid now shows only slots where BOTH people are free (intersection).
6. Tap the first name again to deselect.
7. Verify: grid returns to full heatmap view.
Expected: Instant visual response (150ms transition). No page reload.

**Test 4 — Best time callout (HEAT-05)**
1. Check the "Best time" section above the grid.
2. Verify: it shows a date, time range, and "X of Y people free".
3. Verify: the section is present even before any participants respond (shows "Waiting for responses").
Expected: Best time callout is always visible. When populated, it is visually distinct from the rest of the page.

**Test 5 — Creator confirm-time flow (HEAT-06)**
1. Open the event page in the browser that originally created the event (has the creator cookie).
2. Verify: the Best time callout has an orange border and "Tap to confirm" text (creator-only).
3. Tap the Best time callout.
4. Verify: a bottom sheet slides up showing the slot details, count, free names, and "Confirm this time" button.
5. Tap "Confirm this time".
6. Verify: the sheet closes. The page now shows an orange "Meeting confirmed" banner with the date and time.
7. Verify: the availability CTA ("Mark your availability") is no longer visible.
Expected: Entire confirm flow completes in one tap + one button press.

**Test 6 — Non-creator cannot confirm**
1. Open the same event page in an incognito window (no creator cookie).
2. Verify: the Best time callout renders but has no orange border and no "Tap to confirm" text.
3. Tap the Best time callout — nothing should happen.
Expected: No confirm sheet appears for non-creator visitors.

**Test 7 — Colorblind check (HEAT-02)**
1. On the event page with heatmap data, use your browser's DevTools or a colorblind simulator (e.g. Chrome DevTools > Rendering > Emulate vision deficiencies > Deuteranopia).
2. Verify: heatmap cells are still distinguishable by intensity/brightness — the amber-to-brown scale should remain readable under deuteranopia (no red-green confusion).
Expected: Cells differ in lightness/brightness, not just hue. The scale is warm amber (not red-green).
