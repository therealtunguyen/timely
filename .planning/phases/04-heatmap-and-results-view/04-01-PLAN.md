Modify the POST handler in src/app/api/events/route.ts to:

1. Generate a creator token using `generateId()` from `@/lib/id` (same nanoid-based utility used everywhere). Import it — it is already imported.

2. Store the token in the DB insert: add `creatorToken: creatorToken` to the `.values({...})` call in step 5.

3. After the successful DB insert (after step 6 for specific dates), set an httpOnly cookie in the response:
   - Cookie name: `timely_creator_${id}`
   - Cookie value: the raw creatorToken string
   - Attributes: `httpOnly: true`, `sameSite: 'lax'`, `path: '/'`, `maxAge: 60 * 60 * 24 * 37` (37 days — matches event expiry)
   - Set via `NextResponse.json({ id }, { status: 201, headers: { 'Set-Cookie': ... } })` OR use the cookies() API

   Use the `Set-Cookie` header approach directly on the `NextResponse` since this is a Route Handler:
   ```typescript
   const response = NextResponse.json({ id }, { status: 201 })
   response.cookies.set(`timely_creator_${id}`, creatorToken, {
     httpOnly: true,
     sameSite: 'lax',
     path: '/',
     maxAge: 60 * 60 * 24 * 37,
   })
   return response
   ```

   Replace the existing `return NextResponse.json({ id }, { status: 201 })` at the end with this response-with-cookie pattern.

The `generateId()` call should happen after the rate limit check passes but before the DB insert, so declare `const creatorToken = generateId()` alongside `const id = generateId()` in step 4.
