Run the following verification steps:

**1. Start the dev server:**
```bash
npm run dev
```
Visit http://localhost:3000

**2. Create a test event:**
- Fill in "Title" (required)
- Optionally add a description
- Select 2-3 specific dates OR choose a date range (toggle between modes)
- Adjust the time window if desired (default 9am-9pm)
- Click "Create event"

Expected: Redirects to /e/[nanoid]/confirm with your event title and a copyable link.

**3. Open the event page:**
Click "View event page →" on the confirm page.

Expected: Event page at /e/[nanoid] shows:
- Event title (large heading)
- Description (if you added one)
- Candidate dates listed (or date range)
- Time window
- "Mark your availability" button

**4. Check mobile layout:**
Open Chrome DevTools → Toggle device toolbar → Set to iPhone 12 Pro (390px width).
Reload both the create form (/) and event page (/e/[nanoid]).

Expected: No horizontal scrollbar. Text readable. Buttons full-width. Calendar fits the screen.

**5. Check OG image:**
Visit: http://localhost:3000/e/[nanoid]/opengraph-image

Expected: A 1200x630 PNG renders with the event title and "Mark your availability" text on a warm #FAF8F5 background.

Optionally, paste the event URL at https://www.opengraph.xyz to preview how it will appear in iMessage/WhatsApp.

**6. Test rate limiting (if Upstash is configured):**
Open the browser console and run (replace URL as needed):
```javascript
for (let i = 0; i < 12; i++) {
  fetch('/api/events', {
    method: 'POST',
    headers: {'Content-Type': 'application/json'},
    body: JSON.stringify({title:'t',dateMode:'specific_dates',specificDates:'["2026-12-01"]',dayStart:9,dayEnd:17,timezone:'UTC'})
  }).then(r => console.log(i, r.status))
}
```
Expected: Requests 1-10 return 201 (or 422 after 10 successful events are created and rate limit doesn't matter). Request 11+ returns 429.

**7. Verify UTC in database:**
Run `npm run db:studio` and open the events table.
Expected: `created_at`, `expires_at` columns show timestamps with timezone offset (UTC, format 2026-xx-xxTxx:xx:xx.000Z or similar). No local-time-only timestamps.

**8. Test 404 for nonexistent event:**
Visit: http://localhost:3000/e/aaaaaaaaaa (10 random chars that won't exist)

Expected: Next.js 404 page — NOT a 500 error or blank screen.
